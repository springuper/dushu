# 产品规格文档索引

本目录包含所有重要功能的规格文档（Spec），作为开发和维护的权威依据。

## 📋 规格文档列表

### 核心功能

1. **[历史阅读增强 App 产品规格书](./reading-app-spec.md)**
   - 主产品规格书
   - 定义 MVP 级别的用户价值、核心流程与功能要求
   - 包含阅读界面、LLM 问答、内容管理等核心功能

2. **[数据获取与融合规格书](./data-acquisition-and-merge-spec.md)**
   - 数据获取、提取、融合的完整流程和技术规范
   - LLM 驱动的智能融合方案
   - 变更日志和版本管理
   - 审核流程和 API 规范

## 📚 文档组织原则

1. **Spec 文档**：放在 `specs/` 目录，定义功能规格和技术规范
2. **使用指南**：放在 `docs/` 目录，提供操作指南和最佳实践
3. **工具脚本**：放在 `scripts/` 目录，包含脚本和简要说明

## 🔄 文档更新

- 重要功能变更必须更新对应的 Spec 文档
- Spec 文档是权威依据，其他文档应引用 Spec，而非重复定义
- 过时的文档应及时清理
